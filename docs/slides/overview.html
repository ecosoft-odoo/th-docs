Odoo กับบริบทไทย

โมดูล OCA เพื่อช่วยเสริมการทำงานแบบไทยๆ

ทำไมจึงเกิดเอกสารชุดนี้

Odoo คือซอฟต์แวร์ ERP แบบโอเพ่นซอร์ส ผู้ผลิตคือบริษัท Odoo S.A. ประเทศเบลเยี่ยม

เนื่องจาก Odoo S.A. ต้องดูความต้องการจากทั่วโลก และทำซอฟต์แวร์ให้เป็นกลางๆ จึงเน้นผลิตซอฟต์แวร์มาในลักษณะที่เน้นตอบโจทย์การใช้งานทั่วไปของลูกค้าส่วนใหญ่ ซอฟต์แวร์ที่ทาง Odoo S.A. ผลิตผมขอเรียกว่า Core Odoo

สำหรับ ERP ซึ่งเป็นซอฟต์แวร์หลักของธุรกิจ มักมีความต้องการบางส่วนที่ Core Odoo ยังไม่คลอบคลุม (Industry Specific) และความต้องการอีกบางส่วนซึ่งเป็นความต้องการเฉพาะทางของแต่ละประเทศ ซึ่งผมขอเรียกว่า Thai Localization เลยเป็นสิ่งที่ขาดไม่ได้หากต้องการใช้งาน Odoo ให้ได้จริงทั้งระบบ โดยเฉพาะด้านการเงิน เช่นเรื่องเกี่ยวกับภาษีไทย รายงานที่ส่งกรมสรรพากร ลักษณะการทำธุรกิจแบบ SME (เช่นการวางบิลเก็บเช็ค) และอื่นๆ

ทำไมจึงเกิดเอกสารชุดนี้

จึงเป็นเรื่องปกติที่ัรับรู้กันทั่วไปในวงการว่า ถ้าให้ใช้เฉพาะส่วน Core Odoo แล้วก็มักจะใช้งานไปได้ไม่สุดทาง การวางระบบ Odoo ERP จึงต้องการผู้วางระบบที่เข้าใจปัญหานี้ และมีโมดูลเสริมทั้งในส่วน Thai Localization หรือมีความสามารถในการปรับแต่ง Odoo ให้ตอบโจทย์ที่มันเจาะจงกับธุรกิจไทย

ในประเทศไทยมีผู้วางระบบหลายรายที่มีความสามารถและมี Thai Localization Package เป็นของตัวเอง แต่โดยส่วนใหญ่เป็นแบบต่างคนต่างทำและปิดโค้ดไว้ให้บริการลูกค้าของตัวเอง เรายังไม่มี Thai Localization กลางๆที่เปิดเผยสู่สาธรณะที่ทุกคนสามารถหยิบไปทดลองได้ ทั้งนี้ทำให้มีปัญหากับการวางระบบ Odoo อยู่เหมือนกัน

ทำไมจึงเกิดเอกสารชุดนี้

ตัวอย่างเช่น

  • ผู้ว่าจ้างเองจะไม่่รู้ล่วงหน้าว่าโมดูล Thai Localization ของผู้ให้บริการมีหลักการทำงานอย่างไร จนกว่าจะได้ร่วมงานกัน
  • ผู้ให้บริการรายใหม่ๆที่เข้ามาในตลาดการวางระบบ Odoo แม้ว่าอาจะมีความเชี่ยวชาญด้านธุรกิจ ก็ไม่มีทางไปต้องเริ่มต้นใหม่จากการสร้างชุดโมดูลของตัวเอง
  • ต่างคนต่างทำต่างใช้ไม่มีมาตรฐาน ไม่มีที่ให้แจ้งปัญหา ไม่มีความร่วมมือกันเพื่อปรับปรุงให้ดีขึ้น และวิธีการเขียนก็อาจไม่มีคุณภาพ จะรู้ก็เมื่อสายเสียแล้ว

นอกเหนือจากส่วน Thai Localization แล้ว Business Feature อื่นๆที่ตอบโจทย์เฉพาะกลุ่มประเภทธุรกิจก็เช่นกัน การต่างคนต่างทำต่างมีทำให้เกิดปัญหาในแบบเดียวกัน ถ้าผู้ว่างจ้งได้ผู้ให้บริการที่มีฝีมือก็ถือว่าโชคดีไป

ทำไมจึงเกิดเอกสารชุดนี้

อันที่จริงแล้วจุดแข็งของ Odoo ที่เหนือกว่า ERP อื่นๆคือ

  1. ความเป็น Open Source สามารถแก้ไขได้ทุกจุด
  2. Framework ของ Odoo มีความ Modular สูง ปรับแต่งง่าย

ในต่างประเทศที่ใช้ Odoo กันอย่างจริงจังมากกว่าเรา ในอดีตก็มีปัญหาต่างคนต่างทำเช่นกัน หลายๆบริษัทได้ทำโมดูลเสริมซ้ำๆซ้อนกัน ไม่มีมาตรฐานกำกับ (เช่น บั๊กเยอะ อัพเกรดยาก ไม่มีมาตรฐาน)

ในต่างประเทศจึงมีการรวมกลุ่มกันก่อตั้งสมาคม Odoo Community Association ผมขอเรียกว่า OCA ขึ้นมาเพื่อแก้ปัญหาเหล่านี้ องค์กร OCA จะมีหน้าที่สร้างแพลตฟอร์มการทำงานให้มีมาตรฐานร่วมกัน สนับสนุนให้เกิดการแบ่งปันซอร์สโค้ดและความรู้ระหว่างกัน ปัจจุบันได้มีผู้เข้าร่วมกับ OCA เกือบพันคน และสร้างโมดูลคุณภาพสูงหลายพันโมดูล

วางระบบให้ดีขึ้นด้วยโมดูลของ OCA

จากประสบการณ์ เรามอง ERP ว่าเป็นระบบซอฟต์แวร์ขนาดใหญ๋ มีความซับซ้อนและหลากหลายและ One Size Never Fit All การใช้ Core Odoo อย่างเดียวมักไม่เพียงพอสำหรับโปรเจคที่มีความซับซ้อน ไม่ว่าอย่างไรก็ต้องใช้โมดูลเสริมหรือเขียนเพิ่มเติม

ด้วยโมดูลของ OCA ที่มีจำนวนมากและคุณภาพสูง ร่วมกับ Thai Modules ที่ทาง Ecosoft ได้ริเริ่มน่าจะช่วยให้การวางระบบ Odoo กับธุรกิจไทย เป็นไปได้ราบรื่นขึ้น

แนวทางการเขียนเอกสารชุดนี้

  1. เพื่อให้คนที่สนใจนำ Odoo ไปวางระบบได้มีจุดเริ่มต้น โดยเฉพาะในส่วนความต้องการเกี่ยวกับบัญชีไทย และกระบวนการธุรกิจแบบไทยๆ
  2. เขียนเชิงการอธิบาย Use Case โดยอาจเป็นการผสมกันของหลายๆโมดูลในการตอบโจทย์ ไม่ได้เขียนเพื่ออธิบายตัวโมดูล (แต่ละโมดูลจะมี readme ของตัวเองอยู่แล้ว)
  3. เพื่อจุดประกายให้ผู้ที่นำโมดูลเหล่านี้ได้เห็นคูณค่าของการทำงานร่วมกัน และอาจพิจรณามาเข้าร่วมกันทำงานกับ OCA

ทั้งนี้ทางผู้เขียนไม่ได้เจาะจงว่า โมดูลที่นำเสนอเป็นการทำงานที่ดีที่สุดแล้วสำหรับกระบวนการทำงานหนึ่งๆ แต่เป็นเพียงทางออกหนึ่งจากหลายทาง โมดูลทั้งหมดสามารถถูกปรับปรุงให้ดีขึ้นได้ จากการความเห็นและร่วมด้วยช่วยกันของคนที่ได้นำไปใช้
